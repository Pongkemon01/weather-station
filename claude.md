# Weather Station Project

## Hardware Platform

- **MCU**: STM32L476RG (80 MHz Cortex-M4F, 128KB RAM, 1MB Flash)
- **Power**: Solar cell + Baterry
- **Peripherals**
  - I2C1 speed 400kHz
  - I2C2 speed 400kHz
  - SPI1 speed 20Mbps
  - USART1 speed 4800 with DMA enabled
  - USART2 speed 115200 with DMA enabled
  - USART3 speed 115200 with DMA enabled
  - USB full-speed device-only mode
  - SDMMC 4-bit mode with DMA enabled
  - RTC with external 32.768kHz crystal
  - CRC configured for 16-bit ANSI CRC
  - GPIO output bits:
    - PA4 as SPI1 chip select pin (FRAM_CS)
    - PC0 RED LED (LED_RED)
    - PC1 YELLOW LED (LED_YELLOW)
    - PC2 GREEN LED (LED_GREEN)
    - PC3 BLUE LED (LED_BLUE)
    - PC4 active-low push-button (USR_SW1)
    - PC5 active-low push-button (USR_SW2)
  - GPIO input bits:
    - PC6 active-low SD card detect pin (SD_CD)
    - PC7 active-low SD card write protect pin (SD_WP)
- **IO Expansion**: MCP23017 (I2C1, address 0x20) with 16 I/O pins configured as:
  - GPA0 - GPA7 output pins
  - GPB4 - GPB8 output pins
  - GPB0 - GPB3 input pins with internal pull-up
- **Sensors**:
  - BMP390 temperature and pressure sensors (I2C2, address 0x76)
  - SHT45 temperature and humidity sensors (I2C2, address 0x44)
  - SEM228 light PAR sensor (Modbus RTU, address 0x01 through USART1)
  - R66S rain guage (Modbus RTU, address 0x02 through USART1)
- **User Interface**: Connected to the MCP23017 (IO Expansion)
  - LCD 16x2 character display operating in 4-bit data mode
    - D7 - D4: GPA3 - GPA0 (4-bit data bus)
    - E: GPA4 (Operation Enable : Falling-edge triggered)
    - R/W: GPA5 (Read or Write operation : 0 = Write, 1 = Read)
    - R/S: GPA6 (Command or Data : 0 = Command, 1 = Data)
    - BK-ON: GPA7 (Turn-on Back-light)
  - Red LED: GPB7
  - Green LED : GPB6
  - UP switch (active-low): GPB3
  - DOWN switch (active-low): GPB2
  - ENTER switch (active-low): GPB1
  - MENU switch (active-low): GPB0
- **Non-volatile storage**: CY15B116QN F-RAM (2MB x 8-bit) via SPI1
- **Radio**: A7670E LTE Modem (USART3)
- **SD Card**
- **USB Connection**:
  - Device only
  - One CDC device
  - OTG global interrupt is enabled
  - Manage by the TinyUSB library
- **Debug output**: USART2
- **Memory Map**:
  - SRAM: 0x20000000-0x2001FFFF (128KB)
  - Flash: 0x08000000-0x080FFFFF (1MB, bootloader at 0x08000000-0x08007FFF)

## Architecture

- System configuration and some middle-wares are generated from STM32CubeMX
- Folder structure is generated by the STM32PIO application
- FreeRTOS (version 10.3.1) with CMSIS v2
- Hardware abstraction layer isolates all peripheral access
- Use separate FreeRTOS tasks as much as possible
- Separate modules: sensor drivers, radio driver, user interface driver
- USART1 and USART3 operations are performed through the provided USB_SUBSYSTEM
- USART2 is the "printf" output target used for printing debug messages
- SD card should be formatted as FAT or ExFAT filesystem
- Data stored to SD card in CSV-format files

## Operation intervals

- Scan switches and update LCD and LEDs every 20ms
- Read and process sensors every 1s (synchronized with RTC second counter)
- Re-initialize unready peripheral every 1s
- Upload measured data to an HTTPS server everyday at noon and midnight
- Read and process user activities from user interface every 100ms

## Build System & Toolchain

- **Compiler**: GCC ARM Embedded (arm-none-eabi-gcc)
- **Development Environment**: PlatformIO via VS Code
- **Optimization**: -Os for size, -O2 for time-critical functions only
- **Debugger**: ST-Link V2

## Coding Standards

- **Language**: C99 (no C++)
- **Style**: 4-space indents, max 100 chars per line
- **Naming**:
  - Functions: `ModuleName_FunctionName()` (e.g., `Sensor_ReadTemperature()`)
  - Variables: camelCase for local, g_PascalCase for global
  - Macros: ALL_CAPS_WITH_UNDERSCORES
- **Documentation**: All public functions must have Doxygen-style comments
- **Testing**: Unit tests required for all business logic (not HAL)

## Critical Don'ts

- **NEVER** use `malloc()`, `free()`, or any dynamic allocation after initialization
- **NEVER** access peripherals directly - always use HAL functions
- **NEVER** poll sensors - use interrupt-driven reads where possible

## Allowed Libraries

- STM32L4 HAL and LL libraries
- CMSIS (core peripheral access only)
- FreeRTOS (use only provided)
- TinyUSB (use only provided)
- FATFS (use only provided)

## Memory Budget

- Application code: Max 512KB (leave 512KB for bootloader updates)
- RAM usage: Max 96KB (leave 32KB safety margin)

## Testing Strategy

- Unit tests run on host (x86) with mocked HAL
- Integration tests run on target hardware with automated test harness
- Power consumption validated with current meter on every release build
- Coverage target: 80% for application logic, 100% for critical state machines

## Known Issues & Workarounds

- FRAM_CS must be held low before SPI transactions
- LoRa radio requires CS pin held low for 1ms before SPI transactions

## Important Notes for AI Assistance

- When generating code, assume all HAL functions return error codes (HAL_OK = success)
- Generate optimized code to minimize RAM usage while maintaining resonable complexity
- Sensors may fail at any moment. All generating codde must check the reading status
- LTE Modem may lose connection
- SD Card may be absent
- Some FreeRTOS tasks may require higher priority (such as USB tasks). Search the information from the Internet for appropriate priority level
- This project is partially developped. Optimize the existing code has higher priority. If optimization is too costly, rewriting is also possible.
